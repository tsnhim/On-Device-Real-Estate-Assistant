MODEL OPTIMIZATION ANALYSIS SUMMARY
==========================================

Quantization-only findings:
- Best quant-only accuracy: bf16_cast_gpu with Token F1=0.2408.
- Fastest quant-only latency: dynamic_int8_cpu_ptq with mean latency=1325.9 ms.
- Smallest quant-only disk size: dynamic_int8_cpu_ptq with disk size=120.72 MB.
- Interpretation: BF16/FP16 preserve quality better because they keep more numerical information than INT8. INT8 compresses strongly, but the aggressive quantization can hurt generation quality.

Pruning-only findings:
- Best prune-only accuracy: prune_unstructured_attention_l1_10 with Token F1=0.2392.
- Fastest prune-only model: prune_structured_mlp_intermediate_10 with mean latency=34.4 ms.
- Interpretation: unstructured targeted pruning is safer than structured pruning here. Structured pruning can remove whole functional blocks and greatly improve speed, but it can also collapse quality if important heads or MLP dimensions are removed.
- Disk-size caveat: unstructured pruning may not reduce file size unless the model is saved/executed using a sparse-aware format.

Combined pipeline findings:
- Quant-then-prune and prune-then-quant should be judged by whether the second step creates a new Pareto improvement.
- In these results, most stacked methods do not create enough extra value: several keep the same disk size or lose accuracy while increasing memory.
- Interpretation: stacking optimization methods can compound approximation error. It is useful only when the second step improves a deployment bottleneck without a large quality penalty.

Final selection:
- Selected by balanced score: bf16_cast_gpu with score=0.7109.
- The balanced score weights Token F1 most heavily, then ROUGE-L, throughput, latency, and disk size.
- This makes the selected model a quality-preserving deployment choice rather than simply the smallest or fastest model.

Baseline caveat:
- Baseline reference eval_count=2.

- If these counts differ, baseline comparison should be described as illustrative rather than fully apples-to-apples.
